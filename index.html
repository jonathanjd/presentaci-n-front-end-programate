gulpfile.js

                        
                                var gulp = require("gulp");
                                var browserSync = require("browser-sync").create();
                                var sass = require('gulp-sass');
                                var autoprefixer = require('gulp-autoprefixer');

                                gulp.task("serve", function() {
                                  browserSync.init({
                                    server: "./"
                                  });

                                  gulp.watch("./*.html").on("change", browserSync.reload);
                                });

                                gulp.task('sass', function() {
                                  gulp.src('./sass/*.scss')
                                      .pipe(sass())
                                      .pipe(autoprefixer({
                                          versions: ['last 2 browsers']
                                      }))
                                      .pipe(gulp.dest('./css'));
                                });

                                gulp.task('watch', function(){
                                  gulp.watch(['./sass/*.scss'], ['sass']);
                                });

Arquitectura CSS

SMACSS

Qué es SMACSS?

Desarrollado por Jonathan Snook, el método SMACSS consiste en organizar las reglas CSS en cinco categorías

Categorías

  • Básico
  • Maquetación/Disposición
  • Módulo
  • Estado
  • Tema

Básico

Los estilos por defecto. Dentro de ellos están casi exclusivamente selectores de etiqueta, aunque a veces también se incluyen atributos, pseudoelementos, selectores hijos o padres. Se puede decir que es lo que habitualmente usamos como reset.css o normalize.css.

Maquetación

Divide el contenido en secciones, entendiendo que dentro de cada sección el contenido se dividirá en módulos. Serían las reglas para contenedores, rejillas, etc.

Módulo

Aquellos componentes que pueden ser reutilizados.

Estado

Las reglas que nos marcan el estado de nuestros módulos. Por ejemplo, el estado activo de un botón. También se incluyen dentro de esta categoría los estados de un módulo que tiene diferente aspecto según el lugar en el que se encuentre dentro del site.

Tema

Es una categoría opcional. Incluiría los estilos más visuales dentro del proyecto, que pueden estar ya incluidos dentro de las otras clases, pero que por las necesidades del proyecto, se agrupan dentro de un tema para su mejor uso.

Metodología

BEM

Qué es BEM?

Desarrollada por la empresa rusa Yandex, es una metodología que busca el desarrollo de proyectos de una manera rápida y cuyo mantenimiento sea más sencillo, por muchos años.

Lo primero que has de saber de BEM es que no utiliza etiquetas HTML o IDs como selectores, sino que los bloques y los elementos están definidos por selectores de clase.

Bloque

Es el componente lógico y de funcionalidad independiente de una página, el equivalente a un componente en Web Components. La clase para el bloque suele ser corta y específica. Por ejemplo: .header. Los bloques pueden ser anidados dentro de otros bloques.

Bloque HTML

                        
                                <div class="block">...</div>

Bloque CSS

                        
                            .block { color: #042; }

Elemento

Son las partes del bloque que cumplen una función sólo dentro de ellos. La clase para el elemento lleva el nombre del bloque seguido tras dos guiones bajos, del nombre del elemento. Por ejemplo: .header__logo.

Elemento HTML

                        
                                <div class="block">
                                    ...
                                    <span class="block__elem"></span>
                                </div>

Elemento CSS

                        
                            .block__elem { color: #042; }

Modificador

Nos permite crear un bloque o elemento similar a otro existente tanto en apariencia como en comportamiento. La clase para el modificador está compuesta por la del elemento seguida de dos guiones y el nombre del modificador. Por ejemplo: .header__logo--active.

Modificadores HTML

                        
                                <div class="block block--mod">
                                    ...
                                </div>
                                <div class="block block--size-big block--shadow-yes">
                                    ...
                                </div>

Modificadores CSS

                        
                            .block__elem--mod { }
                            .block--hidden { }
